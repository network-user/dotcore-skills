# Подагенты: веер, схема, проверка

Аудит естественно делится на независимые измерения - **запускай подагентов параллельно**. Это правило, а не опция: один проход линейно по всему репозиторию и медленнее, и раздувает контекст сырыми выдержками кода.

## Когда веером, когда нет

| Уровень / случай | Стратегия |
|------------------|-----------|
| Поверхностный | Линейно, без подагентов (минуты, греп-скан) |
| Репозиторий < ~20 файлов | Линейно даже на среднем - веер не окупится |
| Средний | 2-4 подагента по измерениям |
| Полный | 6-10 подагентов по измерениям и модулям + проверочный проход |
| Фокус на один трек | Веер только внутри этого трека |

Делегируй подагенту, когда: измерение независимо от других; поиск займёт 3+ запросов инструментов; результат - большой объём выдержек, который незачем тянуть в основной контекст. Не делегируй, когда шаг зависит от предыдущего или задача - один греп.

## Деление на измерения

Одно измерение = один подагент. Измерения независимы, поэтому идут в одном сообщении параллельно.

**Трек A (утечки):**
- `leaks-secrets` - секреты/ключи/токены по рабочему дереву
- `leaks-history` - история git (только полный)
- `leaks-pii` - PII, внутренние хосты/IP, machine-paths

**Трек B (код):**
- `code-injection` - инъекции (SQL/command/template/NoSQL)
- `code-exec` - eval/exec, десериализация
- `code-authz-crypto` - auth/authz/IDOR, криптография, небезопасные дефолты
- `deps-supplychain` - зависимости, lockfile, postinstall
- `infra-ci` - Dockerfile, CI (только полный)

На полном уровне крупный трек B можно дополнительно делить **по модулям** (`apps/`, `packages/`, `services/`) - подагент на модуль × категория.

## Схема возврата (structured)

Каждый подагент возвращает **только данные** - массив находок, без человеко-ориентированного текста:

```json
{
  "dimension": "code-injection",
  "findings": [
    {
      "severity": "high",
      "category": "sql-injection",
      "file": "src/api/users.py",
      "line": 42,
      "title": "Конкатенация ввода в SQL-запрос",
      "evidence": "query = \"… WHERE id=\" + re…23  (маска)",
      "source_to_sink": "request.args['id'] -> cursor.execute(...)",
      "recommendation": "Параметризованный запрос / ORM",
      "confidence": "high"
    }
  ]
}
```

- `severity`: critical | high | medium | low | info
- `evidence`: **маскированная** выдержка. Значение секрета не выводить - первые/последние 2 символа, остальное `…`.
- `confidence`: насколько уверенно (для приоритета на проверке).

## Синтез

Собери массивы всех подагентов, **дедуп** по `file + line + category`, отсортируй по severity. Дубли из разных измерений - одна находка с максимальной severity.

## Adversarial-проверка (средний и полный)

Перед вердиктом проверь каждую находку **Critical/High** - срезает false positives:

1. На находку запусти отдельного подагента-скептика с задачей **опровергнуть**: «Вот заявленная уязвимость X в file:line. Найди причину, по которой это НЕ эксплуатируется (санитизация выше по потоку, не достижимо, не тот язык/фреймворк, ввод доверенный). По умолчанию считай опровергнутым, если путь source→sink не подтверждается.»
2. Полный уровень - 2-3 скептика с разными линзами (достижимость, санитизация, реальный impact); решение большинством.
3. Опровергнуто - понизь severity или отбрось; подтверждено - оставь с пометкой «verified».

Не выдавай PASS/FAIL и тем более бейдж по непроверенным Critical/High.

## По агентам

- **Claude Code** - инструмент Task/Agent; для read-heavy поиска - подагент Explore. Параллельные вызовы **в одном сообщении**; при желании `run_in_background` для долгих измерений. Это основной режим.
- **Cursor** - примитива подагентов как в Claude нет. Гоняй измерения **последовательно** в один проход (или background-агентами, если доступны в версии), сохраняя ту же схему находок, синтез и проверочный шаг. Результат тот же, просто без параллелизма.
- **Codex / прочие** - последовательный fallback, схема находок и проверка не меняются.

Независимо от агента итог одинаков: структурированные находки → дедуп → проверка Critical/High → вердикт ([report.md](report.md)).
