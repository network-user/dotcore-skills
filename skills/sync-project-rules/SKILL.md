# Sync Project Rules

Лёгкий скилл DotCore: **дополняет правила для агентов** (`AGENTS.md` + нативные rule-файлы) фактами из репозитория, не трогая README, обложку, бейджи, LoC и аудит. Для частого случая «изменились команды или структура, надо синхронизировать `AGENTS.md`» - без тяжёлого `generate-readme`.

Два главных правила:

1. **Читай репозиторий, не выдумывай.** Источник правды для фактов (команды, пути, зависимости, runtime) - `package.json`, `Makefile`, `pyproject.toml`, `docker-compose.yml`, CI, код.
2. **Дополняй, не переписывай.** Существующие `AGENTS.md`/`CLAUDE.md`/`.mdc`/`GEMINI.md` - это правила, которые написал автор проекта. Скилл их **не переписывает и не реструктурирует**: он добавляет недостающий DotCore-блок и точечно чинит только те факты, что доказуемо расходятся с кодом. Всё остальное остаётся как есть.

Связь с `generate-readme`: этот скилл владеет шаблонами правил. `generate-readme` вызывает его на шаге 5 (правила), а сам отвечает за README, обложку, бейджи, LoC, лицензию и аудит. README этот скилл **не трогает**.

## Когда применять

- Изменились команды/скрипты, модули, зависимости, runtime или структура - синхронизировать `AGENTS.md`
- Дополнить `.cursor/rules/dotcore-project.mdc`, `CLAUDE.md`, `GEMINI.md` DotCore-правилами
- Запросы: «обнови AGENTS.md», «синхронизируй правила проекта», «обнови правила агентов»
- Как шаг правил внутри `generate-readme`

Нужен полный README (обложка, бейджи, пересчёт LoC, аудит) - это `generate-readme`, не этот скилл. Здесь README остаётся как есть.

## Язык и тон

- **Русский**, если проект не целиком на английском. Пакеты и техтермины - как в коде.
- Тон сухой, internal doc. `AGENTS.md` и README - один язык; `.mdc`/`CLAUDE.md`/`GEMINI.md` - кратко на том же языке.
- Добавляемый текст подгоняй под стиль уже написанного в файле - не ломай тон и оформление автора.

## Workflow

### 1. Scan (только rules-relevant)

| Категория | Где смотреть |
|-----------|--------------|
| Runtime | `engines`, `.nvmrc`, `pyproject.toml`, `go.mod`, `Cargo.toml` |
| Команды | `scripts`, `Makefile`, `pyproject` scripts, CI `.github/workflows/` |
| Зависимости | `dependencies`, `devDependencies`, `requirements` |
| Окружение | `docker-compose.yml`, `Dockerfile`, `.env.example` (имена, не значения) |
| Тесты / lint | CI, `pytest.ini`, `eslint.config`, `vitest.config` |
| Структура | `apps/`, `packages/`, `src/` - модули по факту |
| Существующие правила | `AGENTS.md`, `CLAUDE.md`, `.cursor/rules/`, `GEMINI.md` |

Спорное - проверь по коду. README **не читаем как авторитет** и не правим. По итогам Scan запиши две вещи: какие факты в существующих правилах доказуемо расходятся с кодом (кандидаты на точечную правку) и какого DotCore-блока не хватает (кандидаты на добавление).

### 2. Determine launching agent

Определи, из какого агента запущен скилл, и **обязательно** создай или дополни его нативный rule-файл (таблица в [templates.md](templates.md)); если папки нет - создай. `AGENTS.md` затрагивается всегда, независимо от агента.

### 3. Дополни AGENTS.md (additive)

- **Файла нет** - создай по шаблону из [templates.md](templates.md): профиль, быстрый старт, таблица команд, структура, соглашения, env, «что делать / чего не делать», правило README-sync.
- **Файл есть** - **не переписывай**. Сделай минимальный additive-diff:
  - **добавь** недостающие DotCore-блоки (правило README-sync, указатель на стандарт DotCore, ссылку на rule-файл агента запуска), если их нет;
  - **точечно почини** только те факты, что доказуемо расходятся с репозиторием (команда, которой больше нет в `package.json`; переименованный путь; ушедшая зависимость) - минимальной правкой строки;
  - **не трогай** структуру, формулировки, порядок и авторские секции, которые не противоречат коду.

Принцип слияния - в разделе ниже.

### 4. Дополни rule-файлы

`.cursor/rules/dotcore-project.mdc` (`alwaysApply: true`), `CLAUDE.md` (обёртка → AGENTS.md), `GEMINI.md` - по [templates.md](templates.md). Только те, что нужны (агент запуска + уже существующие в репо). Тот же additive-подход: нет файла - создай из шаблона; есть - добавь недостающий DotCore-блок и ссылку на AGENTS.md, не переписывая остального.

### 5. Embed README-sync rule

В `AGENTS.md`, `CLAUDE.md`, `.mdc` встрой (если ещё нет) правило: при **глобальных** изменениях (команды, модули, зависимости, архитектура, runtime) агент обновляет README через `generate-readme`. Определения глобального/не-глобального - в [templates.md](templates.md).

## Дополняй, не переписывай (критично)

Существующий `AGENTS.md` (особенно богатый: backend, i18n, security, доменные правила) **не переписывай и не реструктурируй под шаблон**. Шаблон из [templates.md](templates.md) - только для случая, когда файла ещё нет, и как чеклист того, какие DotCore-блоки должны присутствовать.

Если файл уже есть:

- сохраняй авторский текст, порядок и секции дословно;
- добавляй только недостающие DotCore-блоки;
- факт правь точечно и только при доказанном расхождении с кодом - одной строкой, не переписывая абзац;
- ничего не удаляй из авторских правил;
- сомневаешься, расходится факт или нет, - оставь как есть.

Уникальные правила из старого `CLAUDE.md`/`.cursorrules` не теряй; переносить их в `AGENTS.md` - только если пользователь об этом просит. Таблица миграции - в [templates.md](templates.md).

## Self-check

- [ ] Существующие правила сохранены: структура, формулировки и авторские секции не переписаны
- [ ] Добавлены только недостающие DotCore-блоки; правки фактов - точечные и обоснованы кодом
- [ ] `AGENTS.md` синхронизирован; изменённые команды сверены с `package.json`/`Makefile`/CI (не выдуманы)
- [ ] Rule-файл агента запуска создан или дополнен; папка создана, если её не было
- [ ] `AGENTS.md`/`CLAUDE.md`/`.mdc` содержат правило README-sync
- [ ] `.cursor/rules/dotcore-project.mdc` существует с `alwaysApply: true` (если Cursor в репо)
- [ ] `CLAUDE.md` ссылается на `AGENTS.md`, не дублирует его
- [ ] README **не тронут**; обложка, бейджи, LoC, лицензия не менялись
- [ ] Числа, пути, версии - только из репозитория
- [ ] Diff минимальный: нет переписанных «под шаблон» секций, которые и так были корректны
