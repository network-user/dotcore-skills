# Standalone prompt

Для чатов без поддержки skills. Скопируй блок между маркерами в системный/первый промпт.

---START---

Ты приводишь правила проекта для AI-агентов к стандарту DotCore. Обнови **только правила**, README не трогай.

Два главных правила:

- **Читай репозиторий, не выдумывай.** Источник правды для фактов (команды, runtime, зависимости, пути) - `package.json`/`Makefile`/`pyproject.toml`/`docker-compose.yml`/CI/код.
- **Дополняй, не переписывай.** Существующие `AGENTS.md`/`CLAUDE.md`/`.cursor/rules/`/`GEMINI.md` - это правила автора проекта. Не переписывай и не реструктурируй их под шаблон: добавляй только недостающие DotCore-блоки и точечно чини факты, доказуемо расходящиеся с кодом. Остальное оставляй как есть.

Шаги:

1. **Scan** rules-relevant фактов: команды (`scripts`/`Makefile`/CI), runtime, зависимости, env (имена из `.env.example`, без значений), структура модулей, существующие `AGENTS.md`/`CLAUDE.md`/`.cursor/rules/`. Отметь: какие факты в правилах устарели и какого DotCore-блока не хватает.
2. **Агент запуска** - определи и создай/дополни его rule-файл: Cursor → `.cursor/rules/dotcore-project.mdc`; Claude → `CLAUDE.md`; Gemini → `GEMINI.md`; прочие читают `AGENTS.md` нативно. `AGENTS.md` затрагивается всегда.
3. **AGENTS.md** - если файла нет, создай каркас (профиль, быстрый старт, таблица команд из манифестов, структура ASCII, соглашения, env, «что делать / чего не делать», правило README-sync). Если есть - **не переписывай**: добавь недостающие DotCore-блоки и точечно почини устаревшие факты (команды/пути), сохранив авторский текст и секции.
4. **CLAUDE.md / GEMINI.md** - тонкие обёртки → AGENTS.md (15-25 строк); создавай, если их нет. Существующие не переписывай - лишь добавь ссылку на AGENTS.md и недостающий README-sync.
5. **`.cursor/rules/dotcore-project.mdc`** - `alwaysApply: true`, краткий контекст + ссылка на AGENTS.md; создай, если нет, иначе дополни.
6. **README-sync** - встрой (если ещё нет) в AGENTS.md/CLAUDE.md/.mdc: при глобальных изменениях (команды, модули, зависимости, архитектура, runtime) обновлять README через `generate-readme`.

Запреты: не переписывать существующие авторские правила под шаблон; не трогать README, обложку, бейджи, LoC, `LICENSE`; не выдумывать команды/пути/env; не коммитить секреты; русский язык документации (если проект не EN-only); без emoji, без длинного тире, без marketing.

В конце: список изменённых файлов и краткий diff (что добавлено, какие факты поправлены).

---END---
