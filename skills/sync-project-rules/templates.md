# Шаблоны правил проекта

Шаблоны ниже - для случая, когда файла ещё **нет**: тогда создай его из шаблона по данным Scan. Если файл уже есть - **не переписывай его под шаблон**: добавь только недостающие DotCore-блоки и точечно почини факты, расходящиеся с кодом (см. [SKILL.md](SKILL.md) «Дополняй, не переписывай»). Источник правды для фактов - код и манифесты; для авторских правил - сам существующий файл.

## Разделение ролей

| Файл | Для кого | Содержание |
|------|----------|------------|
| `AGENTS.md` | все coding-агенты | build, test, lint, conventions, запреты, указатели |
| `.cursor/rules/dotcore-project.mdc` | Cursor | краткий контекст + ссылка на AGENTS.md |
| `CLAUDE.md` | Claude Code | тонкая обёртка → AGENTS.md |
| `GEMINI.md` | Gemini CLI | тонкая обёртка → AGENTS.md |
| Rule-файл агента запуска | агент, из которого запущен скилл | его нативный файл (таблица ниже), создать если нет |

README - не зона этого скилла. Длинные command-листы и operational-инструкции для агентов - в `AGENTS.md`.

## Агент запуска → файл правил

Определи, из какого агента запущен скилл, и **обязательно** создай/обнови его нативный rule-файл; нет папки - создай. Канонический `AGENTS.md` пишется всегда.

| Агент запуска | Создать/обновить | Папка (создать, если нет) |
|---------------|------------------|---------------------------|
| Cursor | `.cursor/rules/dotcore-project.mdc` | `.cursor/rules/` |
| Claude Code | `CLAUDE.md` | корень |
| Gemini CLI | `GEMINI.md` (+ `.gemini/` если используется) | корень / `.gemini/` |
| Codex / Copilot / Jules / Aider | `AGENTS.md` (нативно, уже есть) | корень |
| OpenCode / Goose / Roo / Junie / Amp | `AGENTS.md` (нативно) | корень |

Большинство агентов читают `AGENTS.md` нативно - отдельный файл нужен только Cursor (`.mdc`), Claude (`CLAUDE.md`) и Gemini (`GEMINI.md`). Не плоди дубликаты.

## README-sync (глобальные изменения)

В `AGENTS.md`, `CLAUDE.md`, `GEMINI.md` и `.mdc` встрой правило: **при глобальных изменениях функционала** агент обновляет README через скилл `generate-readme` (пересчёт LoC, команды, стек, архитектура) и синхронизирует `AGENTS.md` через этот скилл (additive).

- **Глобальные** (обновлять README): новые/удалённые команды или скрипты, новый/удалённый модуль или пакет, смена архитектуры, изменение зависимостей/стека, новая/убранная фича, смена runtime, переименование точек входа.
- **Не глобальные** (README не трогать): опечатки, внутренний рефактор без изменения команд/API, правки комментариев, мелкие багфиксы.

## Шаблон `.cursor/rules/dotcore-project.mdc`

Имя файла фиксировано: `dotcore-project.mdc`. При конфликте с другими rules - **обнови этот файл**, не создавай дубликат.

```markdown
---
description: DotCore контекст проекта - сборка, тесты, соглашения. Читай AGENTS.md для деталей.
globs:
  - "**/*"
alwaysApply: true
---

# DotCore Project Context

Канонические инструкции: [AGENTS.md](../../AGENTS.md) в корне репозитория.

## Кратко

- **Тип:** {project_type}
- **Установка:** `{install_cmd}`
- **Dev:** `{dev_cmd}`
- **Тесты:** `{test_cmd или нет}`

## Правила

- Источник правды - код, не устаревший README.
- README в стандарте DotCore: без marketing, без `<details>`, без centered hero.
- При глобальных изменениях (команды, модули, зависимости, архитектура) обнови README через `generate-readme`, правила через `sync-project-rules`.
- Минимальный scope изменений.

## README

Человеческая документация: [README.md](../../README.md). Не дублируй её целиком в чат.
```

Тонкая обёртка (15-25 строк). **AGENTS.md - единый источник правды.**

```markdown
# Claude Code

## Триггеры
```

## Миграция и слияние

| Ситуация | Действие |
|----------|----------|
| Есть `.cursorrules` (legacy) | Перенеси уникальные правила в `AGENTS.md` |
| Есть `CLAUDE.md` с контентом | Слей уникальное в AGENTS.md или оставь секцию «Дополнительно» в CLAUDE.md |
| Есть `AGENTS.md` богатый/устаревший | Дополни недостающими DotCore-блоками; устаревшие факты - точечно; структуру и секции **не переписывай** |
| Нет `.cursor/` (а агент - Cursor) | Создай `.cursor/rules/dotcore-project.mdc` |
| Monorepo с пакетами | Один AGENTS.md в корне; вложенные - только если уже есть |
