# Правила проекта для агентов

При каждом запуске скилла **перегенерируй** файлы ниже из данных Scan. Не латай старые версии: fresh regeneration, как в project-documenter. Источник правды - код и манифесты, не предыдущий `AGENTS.md`.

## Разделение ролей (стандарт agents.md)

| Файл | Для кого | Содержание |
|------|----------|------------|
| `README.md` | человек-разработчик | DotCore internal doc: запуск, команды, стек, архитектура |
| `AGENTS.md` | все coding-агенты | build, test, lint, conventions, запреты, указатели |
| `.cursor/rules/dotcore-project.mdc` | Cursor | краткий контекст + ссылка на AGENTS.md |
| `CLAUDE.md` | Claude Code | тонкая обёртка → AGENTS.md |
| Rule-файл агента запуска | агент, из которого запущен скилл | его нативный файл/папка (таблица ниже), создать если нет |
| `LICENSE` + футер `## Лицензия` | любой, кто открыл репо | строгий All Rights Reserved (см. [license.md](license.md)) |
| `docs/portfolio-draft.md` | DotBioSite | только если `audience=portfolio` |

README остаётся сухим техдоком. Операционные инструкции для агентов - в `AGENTS.md`, не дублируй длинные command-листы в README.

## Порядок записи (шаг 5 workflow)

1. `AGENTS.md` - канонический файл
2. **Rule-файл агента, из которого запущен скилл** - создай файл и папку, если их нет (таблица «Агент запуска» ниже)
3. `.cursor/rules/dotcore-project.mdc` - создай `.cursor/rules/` если нет
4. `CLAUDE.md` - обёртка
5. `LICENSE` + футер `## Лицензия` в README - см. [license.md](license.md)
6. `docs/portfolio-draft.md` - если `audience=portfolio`

Codex, Jules, Aider, Copilot читают корневой `AGENTS.md` нативно - отдельный файл для Codex не нужен.

## Агент запуска → файл правил

Определи, из какого агента запущен скилл (по окружению/контексту), и **обязательно** создай или обнови его нативный файл правил; если папки нет - создай её. Канонический `AGENTS.md` пишется всегда, независимо от агента.

| Агент запуска | Создать/обновить | Папка (создать, если нет) |
|---------------|------------------|---------------------------|
| Cursor | `.cursor/rules/dotcore-project.mdc` | `.cursor/rules/` |
| Claude Code | `CLAUDE.md` | корень |
| Codex / Copilot / Jules / Aider | `AGENTS.md` (нативно, уже есть) | корень |
| Gemini CLI | `GEMINI.md` (+ `.gemini/` если используется) | корень / `.gemini/` |
| OpenCode / Goose / Roo / Junie / Amp | `AGENTS.md` (нативно) | корень |

Большинство агентов читают `AGENTS.md` нативно - отдельный файл нужен только Cursor (`.mdc`), Claude (`CLAUDE.md`) и Gemini (`GEMINI.md`). Не плоди дубликаты для агентов, которым хватает `AGENTS.md`.

## README-sync (глобальные изменения)

В `AGENTS.md`, `CLAUDE.md` и `.mdc` встрой правило: **при глобальных изменениях функционала** агент параллельно обновляет README через скилл `generate-readme` (пересчёт LoC, команды, стек, архитектура) и перегенерирует `AGENTS.md`.

- **Глобальные** (обновлять README): новые/удалённые команды или скрипты, новый/удалённый модуль или пакет, смена архитектуры, изменение зависимостей/стека, новая/убранная фича, смена runtime, переименование точек входа.
- **Не глобальные** (README не трогать): опечатки, внутренний рефактор без изменения команд/API, правки комментариев, мелкие багфиксы.

## Что делать агенту

- Перед правками прочитай затронутые файлы и соседний код.
- После изменений запусти релевантные тесты/lint из таблицы выше.
- **README-sync:** при глобальных изменениях функционала (новые/удалённые команды, модули, зависимости, смена архитектуры или runtime) обнови `README.md` и `AGENTS.md` через скилл `generate-readme` - в том числе пересчёт LoC. Мелкие правки (опечатки, внутренний рефактор) README не трогают.
- Не латай разметку README вручную - перегенерируй скиллом.
- Минимальный diff - не рефактори несвязанный код.
- Числа, пути, версии - только из репозитория.

Тонкая обёртка (15-25 строк). **AGENTS.md - единый источник правды.**

## Миграция и слияние

| Ситуация | Действие |
|----------|----------|
| Есть `.cursorrules` (legacy) | Перенеси уникальные правила в `AGENTS.md`, не копируй в README |
| Есть `CLAUDE.md` с контентом | Слей уникальное в AGENTS.md или оставь секцию «Дополнительно» в CLAUDE.md |
| Есть `AGENTS.md` устаревший | Перегенерируй целиком из Scan |
| Нет `.cursor/` | Создай `.cursor/rules/dotcore-project.mdc` |
| Monorepo с пакетами | Один AGENTS.md в корне; вложенные AGENTS.md - только если уже есть в subpackages (не создавай без запроса) |
